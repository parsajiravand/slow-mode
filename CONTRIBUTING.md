# Contributing to Slow Mode

Thanks for your interest. We’re in early stage and welcome clear, focused contributions.

---

## Contributor roles

| Role | Focus | Examples |
|------|--------|----------|
| **Core maintainer** | Product direction, roadmap, releases | Prioritization, releases, community decisions |
| **Infra** | CI/CD, hosting, security | GitHub Actions, deployment, secrets, monitoring |
| **DX / docs** | Developer experience and documentation | README, Quick Start, API docs, tutorials |
| **Device profile contributors** | Device-specific behavior | New device profiles, platform quirks, testing |

You don’t need a formal title to contribute in any of these areas.

---

## How to contribute

1. **Open an issue** – idea, bug, or question.
2. **Discuss** – we’ll align on scope and approach.
3. **Fork, branch, PR** – one logical change per PR; link the issue.
4. **Code/docs** – follow existing style; keep PRs reviewable (not huge).
